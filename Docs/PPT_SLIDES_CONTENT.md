## 📑 슬라이드 4: 개발 동기 (Motivation)

### 제목
**왜 ECG 인증인가?**

### 기존 생체 인증의 한계
| 방식 | 문제점 |
|------|--------|
| 지문 | 실리콘 복제 가능 ❌ |
| 얼굴 | 사진으로 우회 가능 ❌ |
| 홍채 | 고가 장비 필요 ❌ |

### ECG의 장점
```
✅ 살아있는 사람만 가능 (심장이 뛰어야 신호 발생)
✅ 복제/위조 불가능 (실시간 생체 신호)
✅ 개인마다 고유한 파형 (심장 구조 반영)
✅ 웨어러블로 지속 인증 가능
```

### 시각 자료
- 지문 복제 사진 vs ECG 파형 비교

## 📑 슬라이드 6: 핵심 기술 - ECG 처리 파이프라인

### 제목
**ECG 신호 처리 과정**

### 파이프라인 단계
```
1️⃣ 전처리 (Preprocessing)
   - 베이스라인 제거 (0.5Hz 고역 필터)
   - 고주파 노이즈 제거 (45Hz 저역 필터)
   - 전원 노이즈 제거 (60Hz 노치 필터)

2️⃣ R-peak 검출 (Pan-Tompkins)
   - 밴드패스 필터 (5-15Hz)
   - 미분 → 제곱 → 이동평균
   - 적응형 임계값으로 피크 검출

3️⃣ 특징 추출 (37개+)
   - 형태학적: P, Q, R, S, T 진폭/간격
   - 시간: HRV (SDNN, RMSSD, pNN50)
   - 주파수: FFT, LF/HF 비율
   - 통계: 왜도, 첨도, 엔트로피

4️⃣ 서명 생성
   - 특징 벡터 → 정규화 → 이산화 → SHA-256
```

## 📑 슬라이드 7: 핵심 기술 - Pan-Tompkins 알고리즘

### 제목
**R-peak 검출: Pan-Tompkins (1985)**

### 알고리즘 단계
```
원본 ECG
    ↓
밴드패스 필터 (5-15Hz) → QRS 강조
    ↓
미분 필터 → 기울기 강조
    ↓
제곱 → 양수화 + 증폭
    ↓
이동평균 (150ms) → 덩어리화
    ↓
적응형 임계값 → R-peak 위치
```

### 시각 자료
- 각 단계별 ECG 파형 변화 그래프

### 왜 Pan-Tompkins인가?
- 1985년 IEEE 논문 (표준 알고리즘)
- 실시간 처리 가능
- 높은 정확도

## 📑 슬라이드 8: 핵심 기술 - 유사도 계산

### 제목
**인증: 하이브리드 유사도 알고리즘**

### 기존 방식의 문제
```
❌ 코사인 유사도만 사용 → 다른 사람도 높게 나옴
❌ Z-score 정규화만 사용 → 개인 특성 사라짐
```

### 개선된 하이브리드 방식
```python
유사도 = 0.30 × 원본값 차이      # 개인 특성 보존
       + 0.25 × 코사인 유사도    # 패턴 형태
       + 0.20 × 유클리드 유사도  # 거리 기반
       + 0.25 × 핵심 특징 유사도  # 형태학적
```

### 임계값
- **85% 이상**: 인증 성공 ✅
- **85% 미만**: 인증 실패 ❌

## 📑 슬라이드 9: 문제점 및 해결책 #1

### 제목
**트러블슈팅: 블루투스 데이터 전송 문제**

### 문제
```
❌ 블루투스 연결은 성공
❌ 하지만 ECG 데이터가 안 넘어옴
```

### 원인 분석
```
HC-06 모듈 구조:
┌─────────────────────────────────┐
│  [블루투스 RF] ←→ [UART TX/RX]  │
│     (정상)          (문제!)     │
└─────────────────────────────────┘

→ RF와 UART는 독립적!
→ 연결 성공 ≠ 데이터 전송 가능
```

### 해결책
1. TX/RX 핀 교차 연결 확인
2. 테스트 코드로 전송 확인
3. 모듈 고장 시 교체

## 📑 슬라이드 10: 문제점 및 해결책 #2

### 제목
**트러블슈팅: 인증 정확도 문제**

### 문제
```
❌ 본인 ECG로 등록
❌ 다른 사람 ECG로 로그인 → 성공?! 😱
```

### 원인
```python
# 기존: 너무 관대한 설정
scale = 15.0  # 유클리드 거리 스케일
threshold = 0.80  # 임계값

# Z-score 정규화가 개인 특성 제거
```

### 해결책
```python
# 수정: 엄격하게!
scale = 5.0  # 엄격한 스케일
threshold = 0.85  # 높은 임계값

# 원본 값 차이도 반영 (30%)
# 핵심 특징 별도 비교 (25%)
```
