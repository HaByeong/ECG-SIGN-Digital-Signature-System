# 면접 질문 대답 (짧은 버전): "보안 요소 적용 방식"

## 🎯 2분 요약 답변

---

"이 프로젝트에서는 **5가지 핵심 보안 조치**를 적용했습니다.

**첫째, 데이터 프라이버시 보호**입니다. 원본 ECG 신호를 저장하지 않고, SHA-256 해시로 변환하여 저장합니다. 해시는 일방향 함수이므로 복원 불가능해 데이터베이스 유출 시에도 원본 생체 정보는 보호됩니다.

**둘째, 강력한 인증 임계값**입니다. 코사인 유사도 0.85 이상이어야 인증이 성공하도록 설정하여, 다른 사람이 우연히 통과할 확률을 5-15% 수준으로 낮췄습니다.

**셋째, 세션 관리**입니다. 로그인 시 UUID 기반 세션 ID를 발급하고, 1시간 후 자동 만료되도록 하여 무단 접근을 방지합니다.

**넷째, 스레드 안전성**입니다. threading.Lock을 사용하여 동시 접속 환경에서도 데이터 무결성을 보장합니다.

**마지막, 입력 검증**입니다. 사용자 입력의 유효성을 검증하고, 중복 등록을 방지합니다.

---

**현재 한계점**은 전송 암호화(TLS/SSL)와 저장 암호화가 부족한 점입니다. 
**개선 계획**으로는 TLS/SSL 적용, JWT 토큰 도입, 데이터베이스 암호화를 단계적으로 진행할 예정입니다."

---

## 🎯 1분 초간단 버전

---

"보안은 **3단계**로 적용했습니다.

**1. 데이터 보호**: 원본 ECG를 SHA-256 해시로 변환하여 저장해 프라이버시를 보호합니다.

**2. 인증 강화**: 유사도 임계값 0.85로 설정하여 타인의 인증 성공 확률을 최소화합니다.

**3. 세션 관리**: UUID 기반 세션을 1시간 타임아웃으로 무단 접근을 방지합니다.

**개선 계획**으로는 TLS/SSL 암호화와 데이터베이스 암호화를 추가하겠습니다."

---

## 📌 핵심 키워드 (암기용)

1. **SHA-256 해시** - 원본 미저장, 프라이버시 보호
2. **임계값 0.85** - False Positive Rate 최소화
3. **세션 타임아웃** - 무단 접근 방지
4. **스레드 안전성** - 동시성 보장
5. **입력 검증** - 악의적 입력 방지

**개선**: TLS/SSL, JWT, 데이터베이스 암호화
